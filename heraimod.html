Hera IMOD

Elevating the Hera IMOD lab system with a modern, clean desktop UI. 💻

Details

Role UI Designer
Timeline Apr – May 2024
Skills UI, UX, Figma
Client Hera Laborsysteme

Overview

Hera Laborsysteme's IMOD desktop software ran on a outdated, hard-to-use interface. Working directly with the company, I redesigned the UI building a new component system in Figma, testing against a live demo module, iterating on their feedback, and finishing with a full dark mode. A real client project focused on usability and visual polish.

Problem

An outdated UI was holding the whole system back.

The IMOD desktop software worked, but its interface was visually outdated and inefficient to use. The look was unappealing and, more importantly, it got in the way of the work.

Navigating the system was harder than it needed to be, which led to frustration and slowed users down.

A modern, intuitive redesign was needed to improve usability and overall satisfaction without disrupting the workflows users already relied on.

Design question

How might we modernise the IMOD interface making it cleaner and easier to use while keeping the system familiar to existing users?

Solution

A sleek, modern interface that's both better looking and easier to work with.

The redesigned UI keeps the system's structure intact but gives it a clean, modern aesthetic built on a consistent component library.

Clearer navigation and friendlier controls make tasks faster and less frustrating to complete, so the software fits smoothly into users' daily routines.

The result is a familiar system that simply feels more efficient, more polished, and easier to trust.

Understanding the system

The project started with a meeting with the CEO to understand how the system works and what it needed to do.

I received a demo module to connect to my own PC, so I could test the software hands-on and design against the real interface rather than screenshots.

Research & concept

My first approach was to refresh the aesthetics while keeping the existing layout so the system stayed familiar to its users.

I researched how each component could look and which colours would suit the system: the taskbar drew on how Notion and Google Chrome present icons and hover states, while sliders and buttons took cues from other software, Pinterest, and Dribbble.

[02]

Creating new components

In Figma I designed a fresh set of buttons, sliders, input fields, and other core components making sure each one was both functional and visually consistent.

Consistent system

A unified component library so every screen shares the same look, spacing, and behaviour.

The 4px rule

Spacing and padding built on a 4px grid, keeping alignment tidy and predictable throughout.

Familiar controls

Buttons, sliders, and inputs modelled on patterns users already know from everyday software.

[03]

Implementation & feedback

I integrated the new components with the demo module and sent the updated file to the company for review.

Based on their feedback I iterated on the design, making the changes they needed. This collaborative loop kept the redesign aligned with real user needs and how the system is actually used.

[04]

Final touches: dark mode

On the client's request, I added a full dark mode to the system a modern touch that rounds out the redesign and improves comfort during long sessions.

Both themes share the same component system, so light and dark stay perfectly consistent.

[05]

Reflection

Hera IMOD was my first real client project, and it taught me how design works inside an existing product and an existing team respecting what already works while modernising what doesn't.

Building a consistent component system, holding to a spacing rule, and iterating directly on client feedback sharpened my UI craft and showed me the value of a tight feedback loop.

Delivering a polished light and dark interface that fit smoothly into the company's software was a rewarding way to see a redesign through end to end.
